tapto

Privacy Policy

Last updated: 2026-06-17

tapto turns NFC tags and QR codes into reprogrammable links. It has no backend of its own — your tags live in your own GitHub account.

Data

  • Only your sign-in is stored locally. Your GitHub access token (and the id of your tapto gist) live in chrome.storage.local and clear when you remove the extension. Sign-in uses the GitHub Device Flow (scope gist); the token is stored on your device and sent only to GitHub.
  • Your tags live in your GitHub. The tags you create — names, destination links, and note text — are saved as a single tapto.json gist in your own GitHub account via api.github.com. Your GitHub username is read to label the panel and to form each tag's address.
  • Your tag map is public. The gist is public, so anyone holding one of your tags can read where it points. Keep private destinations (home Wi-Fi, private documents) out of your tags.
  • Resolving a tag. Tags resolve through neves.cloud/tapto, a static page that runs in the visitor's browser: it reads your public gist and forwards. tapto operates no server and receives nothing about who taps a tag.

Not done

No tracking, analytics, cookies, or ads. No data is sold or shared beyond storing your tags in your own GitHub — provider use follows GitHub's privacy statement.
